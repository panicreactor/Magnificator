# Як отримати готовий `Magnificator.vst3` для Windows 11 x64

Збірка відбудеться **в хмарі GitHub** на Windows-сервері з Visual Studio 2022 —
вам не треба нічого встановлювати локально, крім звичайного браузера.

Весь процес — **~10 хвилин**, із них ~5 хвилин чекаєте на білд.

---

## Крок 1 — Створіть безкоштовний акаунт на GitHub

Якщо вже маєте — пропускайте. Інакше: https://github.com/signup

## Крок 2 — Створіть новий (приватний чи публічний) репозиторій

1. Зайдіть на https://github.com/new
2. **Repository name:** `Magnificator` (будь-яке інше теж піде)
3. Залиште все інше як є, натисніть **Create repository**

## Крок 3 — Завантажте файли проєкту

На сторінці нового репозиторію ви побачите напис **«uploading an existing file»**.
Натисніть на нього.

1. Розпакуйте `Magnificator.zip` на комп'ютері
2. Виділіть **усе вміст** папки `Magnificator` (не саму папку, а її вміст:
   `CMakeLists.txt`, `README.md`, `Source/`, `.github/`, `.gitignore` тощо)
3. Перетягніть у вікно GitHub
4. Внизу натисніть **Commit changes**

> ⚠️ **Важливо:** переконайтеся, що завантажилась папка `.github/workflows/`.
> Деякі операційні системи приховують імена, що починаються з крапки.
> На Windows в Explorer: View → Hidden items увімкнути.

## Крок 4 — Запустіть білд

Він стартує **автоматично** після першого пуша.

1. Зайдіть у вкладку **Actions** (угорі сторінки репо)
2. Побачите запис **«Build Magnificator (Windows x64)»** зі статусом жовтий
   кружок (іде збірка) → зелена галочка (готово) або червоний хрестик (помилка)
3. Клікніть на цей запис, щоб побачити логи в реальному часі

Очікуваний час: **3–5 хвилин** (при першому білді JUCE компілюється довше;
наступні запуски будуть швидші завдяки кешу).

## Крок 5 — Завантажте `.vst3`

Коли білд завершиться зеленою галочкою:

1. На сторінці цього workflow run прокрутіть униз до блоку **Artifacts**
2. Клікніть **`Magnificator-VST3-Windows-x64`** — завантажиться zip-файл
3. Також можна завантажити **`Magnificator-Standalone-Windows-x64`** —
   це `.exe`, що запускає плагін як окремий застосунок (для швидкого тесту
   без DAW)

## Крок 6 — Встановіть у систему

1. Розпакуйте `Magnificator-VST3-Windows-x64.zip`
2. Усередині буде папка **`Magnificator.vst3`**
3. Скопіюйте її цілком (як папку!) у:

   ```
   C:\Program Files\Common Files\VST3\
   ```

   Знадобляться права адміністратора.

4. Перезапустіть ваш DAW (FL Studio, Cubase, Reaper, Ableton Live 11+,
   Studio One тощо).
5. Просканувати VST3-папку: у більшості DAW це **Preferences → Plugins → Scan**
6. `Magnificator` з'явиться у списку ефектів (категорія **Fx → Reverb**).

## Типові проблеми

### 🔴 Білд червоний (помилка)

Зайдіть у failed run, відкрийте крок, який червоний. Найчастіші причини:
- **Не завантажилась папка `.github/`** → workflow файла нема → білд не запустився.
  Пересвідчіться через GitHub UI, що `.github/workflows/build-windows.yml` видно.
- **Синтаксична помилка у CMake** → практично неможливо, бо я вже прогнав конфіг.

Скиньте повний лог кроку «Build (Release)» — допоможу розібратись.

### 🟡 DAW не бачить плагін

1. Перевірте шлях — саме `C:\Program Files\Common Files\VST3\Magnificator.vst3\`
   (*папка*, а не файл).
2. FL Studio: Options → Manage plugins → **Find plugins** (не просто Refresh).
3. Cubase/Nuendo: у Plugin Manager натисніть **Refresh** з вимкненим фільтром
   «Blocklist».
4. Ableton Live 11+: Preferences → Plug-Ins → Use VST3 Plug-Ins = ON →
   Rescan VST3 Plug-In Folder.

### 🟡 Плагін завантажується, але GUI не з'являється

Можлива нестача Windows runtime. Я компілюю зі статичним `libstdc++`,
але на дуже старій системі може не вистачити універсального `vcruntime140.dll`.
Якщо у вас Windows 11 — такого не буде; на Windows 10 може знадобитись
**Microsoft Visual C++ Redistributable 2015-2022** (x64):
https://aka.ms/vs/17/release/vc_redist.x64.exe
